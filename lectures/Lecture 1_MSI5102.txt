Feature Mapping
• To learn a predictor, the data records 𝑥, 𝑦 must be mapped to real vectors
• Feature mappings usually work on each component separately:
𝑥 = 𝑥1 , 𝑥2 , … , 𝑥𝑛 → (𝜙1 𝑥1 , 𝜙2 𝑥2 , …, 𝜙𝑛 𝑥𝑛 ) ∈ ℝ𝑠

where 𝜙𝑖 (𝜓𝑗 ) maps a component to a real vector

•

Feature mapping has two purposes:
-- Convert the learning process into numerical computing
-- Put the different data types on an equal footing, i.e., real vectors
Example 1) Each real (or integer) number maps to itself.
2) For a Boolean field, 𝜙 FALSE = −1; 𝜙 TRUE = 1.
3) Color maps to a 3-dimenationl vector (R, G, B).
-- Red maps to (255, 0, 0)
-- Dark red maps to (192, 0, 0)

Feature Engineering
▪ Basic Idea
-- start with some features
-- then process or transform them to produce new (engineered) features
-- use these new features in your predictor
▪ Was it a good idea? did it improve your predictor?
-- train your model with original features and validate performance
-- train your model with new features and validate performance
-- if performance with new features is better, your feature engineering was successful

▪ Calculus provides tools to:

▪ Core uses in ML:
• Model training (gradient descent) for regression models
• Backpropagation for neural networks
• Sensitivity and stability analysis
