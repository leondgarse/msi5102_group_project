for prediction, where
-- internal nodes test (evaluate) an attribute (feature),
-- branches represent the possible values or outcomes
of that test, and
-- leaves output the final predicted class for
a sample instance.

Why DTs could be a good AI choice?
-- output is discrete
-- interpretable
-- no large data available
-- classes are disjoint

Random Forest
 Bagging with decision trees built with extra randomness
 Idea:
1. Given a training data set D
2. For k=1, 2, …, K.
2.1. Draw a new data set 𝐷 of same size
by sampling with replacement from D
2.2 Train a decision tree 𝑇 on 𝐷 by recursively repeating
the following steps
-- At each node, selecting m features uniformly at random,
(without replacement), from all available features
-- Pick the best split feature among the m ones to
split the data samples into two groups
3. Form the final prediction model
•

Regression  average vote for classification

•

Classification  majority vote

Key hyperparameters
 n_estimators: The number of trees in the forest
 max_features (size of random feature subset)
 min_samples_split: the min. no. of training samples required at a node for splitting that node
 min_samples_leaf: the min. no. of training samples that must be present in each leaf node after a split

n_estimators is chosen by validation and trade-off:
-- Few trees → unstable, high variance
-- More trees → performance improves
-- Enough trees → performance plateaus
-- Choose the value where performance stabilizes
